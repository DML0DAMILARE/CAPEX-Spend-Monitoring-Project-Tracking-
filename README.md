# CAPEX Spend Monitoring & Project Tracking

## Overview
This project simulates a real-world financial analytics task at a fast-growing SaaS company. The goal is to design a data-driven CAPEX spend monitoring solution that helps stakeholders regain control over capital investments, improve budget visibility, and align spending with strategic priorities.

## Learning Outcomes
- CAPEX management fundamentals (budget vs. actual, forecasting)
- Data cleaning and validation in Excel / Power Query
- Variance analysis and scenario modeling (what-if analysis)
- Data modeling and DAX in Power BI
- Interactive dashboard design for executive reporting
- Vendor cashflow analysis and working capital insights
- Business communication and stakeholder presentation

## Tech Stack
- **Excel** – data cleaning, variance modeling, scenario analysis, sensitivity checks
- **Power BI** – data modeling, DAX measures, interactive dashboards
- **Markdown / PowerPoint** – documentation and executive reporting

## How to Use This Repository
1. **Explore the Excel models** – review data dictionary, variance calculations, and scenario drivers.
2. **Open the Power BI dashboard** – connect to the provided Excel files (refresh paths if needed). Use slicers to filter by department, project, or scenario.
3. **Review the executive deck** – understand key insights, top overspent projects, and recommendations.
4. **Adapt for your own data** – replace source files with your CAPEX data, update field mappings, and refresh the model.

## Key Metrics & KPIs
- Total Approved Budget vs. Actual Spend
- Variance % (by project, department, category)
- Scenario-adjusted total spend (best / base / worst cases)
- Monthly CAPEX outflows by vendor
- Payment term impact on cashflow timing
